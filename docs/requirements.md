# 需求与范围说明（课程设计 v1）

## 1. 项目目标

本项目用于《应用软件架构课程设计》展示，目标不是构建完整工业级平台，而是在有限周期内完成一个结构清晰、亮点突出、能够完整演示业务闭环的数字孪生试验场管理系统。

最终应清楚体现：

1. 前后端分离架构。
2. 后端分层架构。
3. 主从表业务设计。
4. 达梦 DM8 数据库应用。
5. WebSocket 实时接口。
6. DeepSeek API 调用。
7. 数字孪生监控展示。

---

## 2. 必须完成的功能

## 2.1 登录与角色权限管理

最小要求：

1. 支持登录。
2. 支持根据角色显示不同菜单。
3. 支持后端接口基本鉴权。

## 2.2 试验预约与审批

最小要求：

1. 可创建预约草稿。
2. 可维护多个资源明细。
3. 支持提交、教师审核、主任审批。
4. 审批通过后生成试验任务。

## 2.3 资源设备管理

最小要求：

1. 资源增删改查。
2. 资源状态维护。
3. 预约时能进行资源冲突校验。

## 2.4 数字孪生监控

最小要求：

1. 展示试验场简化场景。
2. 展示模型船位置变化。
3. 展示实时轨迹和关键指标。

## 2.5 实时数据采集与告警

最小要求：

1. 模拟数据周期推送。
2. WebSocket 实时刷新页面。
3. 支持低电量、越界、数据突变等告警。

## 2.6 归档与 AI 分析

最小要求：

1. 查看历史试验任务。
2. 查看轨迹、曲线、告警记录。
3. 调用 DeepSeek API 生成 AI 报告。

---

## 3. 建议完成的增强项

1. 首页驾驶舱统计。
2. 轨迹回放时间轴。
3. 文件上传下载。
4. 审批日志展示。
5. 告警处理闭环。

---

## 4. 明确不作为本期硬性要求的内容

1. 真实硬件接入。
2. 高精度 3D 建模。
3. 真实视频流平台。
4. 复杂工作流引擎。
5. 大规模并发优化。

---

## 5. 演示成功标准

若系统可以完整演示以下链路，则认为课程设计目标基本达成：

1. 学生创建并提交一条带多资源明细的预约。
2. 教师审核通过。
3. 主任审批通过并自动生成试验任务。
4. 监控页展示模拟轨迹和实时数据。
5. 系统触发至少一种告警。
6. 试验完成后可查看归档数据。
7. 可生成并展示一份 AI 分析报告。
