# 答辩演示脚本

按以下顺序操作，可完整展示系统闭环（约 15–20 分钟）。

## 1. 管理员登录

- 账号：`admin` / `123456`
- 展示：左侧菜单包含「用户管理」（其他角色无此菜单）

## 2. 查看首页驾驶舱

- 路径：工作台 `/dashboard`
- 展示：4 个统计卡片、3 个 ECharts 图表、数据库类型标签
- 说明：数据来自后端 API，非前端硬编码

## 3. 学生提交预约

- 退出后用 `student01 / 123456` 登录
- 路径：试验预约 → 新建预约
- 填写试验名称、时间，**添加 2 条以上资源明细**（主从表）
- 保存草稿

## 4. 冲突检测与提交

- 点击「提交」
- 系统先执行资源冲突检测，通过后进入教师审核

## 5. 教师审核

- `teacher01 / 123456` 登录
- 试验预约 → 教师审核 → 通过

## 6. 主任审批

- `director01 / 123456` 登录
- 试验预约 → 主任审批 → 通过
- 打开预约详情，展示审批流程步骤与 **EXP_APPROVAL_LOG**
- 点击「查看生成的试验任务」

## 7. 准备并启动试验

- 试验任务页：准备完成 → 启动试验

## 8. 数字孪生监控

- 路径：数字孪生监控
- 点击「模拟试验开始」
- 展示：模型船移动、轨迹线、实时曲线、WebSocket 状态
- 点击演示按钮触发低电量/越界告警

## 9. 完成试验并归档

- 试验任务页：完成试验 → 归档
- 试验归档页：拖动时间轴 slider 回放轨迹与曲线

## 10. 生成 AI 分析报告

- AI 分析页：选择试验 → 选择分析类型 → 生成报告
- 展示：当前模式（Mock / DeepSeek API）、结构化报告段落

## 11. 达梦数据库验证

- 打开达梦管理工具
- 查看 `EXP_RESERVATION`、`EXP_RESERVATION_RESOURCE` 主从表数据
- 查看 `EXPERIMENT_TASK`、`ALARM_RECORD`、`AI_REPORT` 表数据
- 与页面数据对比一致

## 答辩前一键重置（可选）

```bash
cd backend
python -m scripts.seed_db
```
